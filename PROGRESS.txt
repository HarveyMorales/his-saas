================================================================================
  PATRONES TÉCNICOS CLAVE
================================================================================

DUAL-MODE DATA
  Todas las vistas usan datos reales si tenantId existe, mock si no:
    const isLive = !!tenantId;
    const displayData = isLive ? dbData.map(transform) : MOCK_DATA;

LIVE DB BADGE
  Todas las vistas wired muestran:
    {isLive && <span style="...green badge...">LIVE DB</span>}

REFRESHKEY PATTERN
  Para que un child refetch cuando el parent le indica:
    // Parent: incrementa counter
    setPatientsRefreshKey(k => k + 1);
    // Child prop: refreshKey?: number
    const prevKey = useRef(refreshKey);
    useEffect(() => {
      if (refreshKey !== prevKey.current) { prevKey.current = refreshKey; refetch(); }
    }, [refreshKey]);

SERVER ACTIONS
  Todas las mutaciones van por app/actions/*.ts con "use server"
  Importar en componentes client con: import { action } from "@/app/actions/..."

COLUMNAS CAMELCASE
  Prisma genera camelCase: tenantId, authId, scheduledAt, createdAt
  NUNCA usar snake_case (tenant_id, auth_id, etc.)
